# bmad-idea Operating Modes

Use `bmad-idea` to choose the **smallest framing move that meaningfully reduces ambiguity**.

## Mode selection heuristics

### 1. Problem framing
Choose this when the user keeps describing features, solutions, or ambition but cannot yet state:
- what problem matters,
- for whom it matters,
- what evidence or constraint is shaping the choice.

Good prompts:
- "Help me sharpen this idea before we plan it"
- "I have a bunch of notes but not a real problem statement"
- "What should we clarify before writing a PRD?"

Primary artifact: `problem brief`

Recommended sections:
- target user or situation
- current pain / friction
- desired outcome
- why this matters now
- assumptions to test next

### 2. Audience and value framing
Choose this when the concept exists but the market/user/value angle is weak.

Good prompts:
- "Who is this really for?"
- "Our positioning is generic"
- "What job are we helping with?"
- "How do we explain why this is better?"

Primary artifact: `positioning brief`

Recommended sections:
- primary audience
- JTBD / pains / gains
- differentiated promise
- proof points or missing proof
- message risks / confusion traps

### 3. Concept shaping
Choose this when the user has multiple partial ideas and needs one tighter direction.

Good prompts:
- "Brainstorm and narrow this"
- "Turn these notes into one concept"
- "What should the first version actually be?"

Primary artifact: `concept brief`

Recommended sections:
- concept summary
- problem / user / value
- must-have constraints
- scope boundaries
- open questions

### 4. Game concept framing
Choose this when the packet is still pre-production and centered on player fantasy or core loop.

Good prompts:
- "Shape this game idea"
- "What are the pillars for this concept?"
- "How do we define the hook and loop before a GDD?"

Primary artifact: `game concept packet`

Recommended sections:
- player fantasy
- target audience / platform
- pillars
- core loop
- progression / tone / differentiator
- what the first playable should prove

### 5. Story packaging
Choose this when the concept exists but people do not yet understand it or support it.

Good prompts:
- "Turn this into a pitch"
- "Make the story clearer"
- "We need a concept narrative for stakeholders"

Primary artifact: `story / pitch packet`

Recommended sections:
- audience
- current problem or tension
- desired transformation
- promise
- proof / evidence / unresolved risk
- call to action or next decision

## Anti-patterns
- Do not mix all five modes in one response.
- Do not produce a giant brainstorm transcript as the main artifact.
- Do not jump into roadmap, sprint, or campaign execution detail when the current job is still framing.
- Do not use story packaging when the real gap is still basic problem or audience clarity.
