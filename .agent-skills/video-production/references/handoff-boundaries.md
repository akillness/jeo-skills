# Handoff boundaries

## Keep this skill focused on
- Choosing the production mode
- Shaping the workflow and stack recommendation
- Defining the minimum asset packet
- Naming QA / approval / publishing handoffs

## Route out or de-emphasize when
- The request is only manual creative polish or editor technique critique
- The main problem is ad strategy, launch messaging, or content strategy rather than video execution
- The user only needs a transcript fix, upload action, or scheduling workflow with no production-planning layer

## Practical guidance
- If the user says **Remotion**, keep the canonical workflow but lock the mode to code-first programmable video.
- If the request sounds like **"make 300 variants from a spreadsheet / CRM / product feed"**, bias toward template/API automation unless the user explicitly wants to own the codebase.
- If the ask is **"turn our webinar/podcast into clips every week"**, bias toward hybrid clip / repurposing and call out caption + selection QA.
- If the ask already has generated output and the problem is final polish, call out the manual-finish handoff instead of pretending the entire workflow belongs here.
