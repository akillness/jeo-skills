# Hourly PR Mode Decision

This note documents deterministic mode selection for unattended hourly runs.

## Goal
Provide one reusable decision output that converts open-PR gate state and check status into an execution mode:
- carry-forward on existing PR
- checks-degraded (no new PR)
- new PR eligible

## Script
`python3 .agent-skills/survey/scripts/decide_hourly_pr_mode.py <open_prs.json> <checks_summary.json> <output.json>`

## Inputs
- `open_prs.json`: output of `gh pr list --state open --json ...`
- `checks_summary.json`: generated by `collect_open_pr_checks_artifacts.py`

## Output fields
- `open_pr_count`
- `checks_status`
- `merge_blocked_reason`
- `mode`
- `rationale`
- `open_pr_numbers`

## Current mode taxonomy
- `carry-forward-open-pr`
- `carry-forward-blocked-no-checks`
- `checks-degraded-no-new-pr`
- `new-pr-eligible`

## Why this matters
This keeps hourly blocker behavior consistent across runs and reduces ad-hoc shell branching.
