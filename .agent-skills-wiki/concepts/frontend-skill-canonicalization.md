# Frontend Skill Canonicalization

## Durable finding
The frontend cluster has at least one true canonical-vs-alias pair: `react-best-practices` should own general React/Next.js performance work, while `vercel-react-best-practices` should exist only as a compatibility alias for legacy exact-name workflows.

## Why this matters
- Tier-1 discovery relies heavily on compact skill metadata.
- Public inventory surfaces list frontend skills side-by-side, so duplicated descriptions create avoidable activation noise.
- The best fix is usually canonicalization plus sharper trigger wording, not another new frontend skill.

## Follow-up note
`design-system` and `frontend-design-system` still look like a likely future review target, but they were intentionally left out of this bounded run.
