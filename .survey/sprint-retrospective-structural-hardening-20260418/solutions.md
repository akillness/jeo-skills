# Solution Landscape: sprint-retrospective structural hardening

## Solution List
| Name | Approach | Strengths | Weaknesses | Notes |
|------|----------|-----------|------------|-------|
| Atlassian Team Playbook Retrospective | Facilitation guide / workshop play | Strong framing around effective retros, team improvement, and safety-aware facilitation | Product-agnostic guidance can still become template-heavy if copied directly | Direct page retrieval |
| Scrum.org Sprint Retrospective | Scrum definition / canonical ritual framing | Keeps the ceremony tied to inspecting people, interactions, process, tools, and DoD | Narrow Scrum framing does not solve remote/hybrid or dead-action-item issues by itself | Direct page retrieval |
| Miro retrospective guides | Remote/hybrid collaboration guide + templates | Strong on distributed engagement, async/hybrid variants, and workshop mechanics | Can encourage template-shopping if used without stronger routing | Direct page retrieval |
| Parabol retrospective resources | Remote-team facilitation + idea bank | Strong browser-visible framing on learning, resilience, transparency, and remote formats | Broad list of formats can bloat the front door if copied wholesale into one skill | Browser-rendered primary page evidence |
| TeamRetro template | Action-focused retrospective template | Keeps learnings tied to actions for the next sprint | Template/tool framing alone does not separate retro work from planning or daily coordination | Direct page retrieval |
| Retromat | Format randomizer / template source | Good source of varied retro prompts and anti-staleness ideas | High risk of turning retros into format novelty without follow-through | Existing skill reference / prior repo evidence |

## Categories
- **Canonical ritual framing:** Scrum.org
- **Facilitation and workshop guidance:** Atlassian Team Playbook, Parabol
- **Remote / hybrid board-based facilitation:** Miro, TeamRetro
- **Template / prompt catalogs:** Retromat, Parabol idea collections, TeamRetro templates

## What People Actually Use
In practice, teams still mix lightweight boards/docs, template libraries, and manual action tracking instead of adopting one magical retrospective system. Indexed community titles show the recurring operational questions are still about remote participation and how/where to track action items. That means the repo skill should optimize for intake classification, action follow-through, and truthful route-outs—not for listing more retro games.

## Frequency Ranking
1. Remote/hybrid retrospective facilitation and engagement
2. Turning observations into a small set of owned actions
3. Avoiding stale repeated retro formats
4. Distinguishing retrospectives from planning / estimation / standup work
5. Handling milestone/postmortem variants without deep incident-RCA sprawl

## Key Gaps
- Most solution surfaces over-index on format variety compared with action follow-through.
- Remote and hybrid facilitation advice exists, but it is often packaged as template catalogs rather than a clear mode-selection router.
- Few public guides sharply separate retrospective work from adjacent PM rituals.
- Milestone/game/postmortem variations are usually bolted on, not treated as first-class route-in cases.

## Contradictions
- Marketed claim: more retrospective formats keep the ritual fresh.
- User reality: teams still ask how to get participation and how to make action items survive.

- Marketed claim: one retrospective tool can cover the workflow end to end.
- User reality: teams keep falling back to docs, boards, spreadsheets, or ad hoc trackers for follow-through.

## Key Insight
The best bounded improvement is not another retrospective skill or another template bundle. It is to harden `sprint-retrospective` into a smaller routing-first anchor that picks one facilitation mode, reviews prior commitments first, constrains action count, and routes planning / estimation / daily-sync work outward. That keeps the PM cluster clean while matching the actual workflow evidence better than a bigger template catalog would.
