# Solution Landscape: steam-store-launch-ops

## Solution List
| Name | Approach | Strengths | Weaknesses | Notes |
|------|----------|-----------|------------|-------|
| Steamworks docs | Official platform documentation | Canonical for store page, trailers, demos, wishlist reporting, visibility rounds, UTM analytics | Documentation is broad and not packaged as an operator checklist | Direct source of truth for required Steam surfaces |
| How To Market A Game | Education / playbook content | Strong heuristic guidance for wishlist funnels, festivals, streamers, press, and Steam-first launch thinking | Advice is content-heavy rather than a reusable execution scaffold | High-signal field guidance used by indie devs |
| presskit.gg field guides | Workflow guides + support tooling | Current, tactical, focused on page optimization and Next Fest readiness | Not a local reusable skill; still requires manual translation into workflow steps | Covers both page optimization and fest preparation |
| IMPRESS | Checklist + tooling suite | Practical launch/fest checklist plus supporting tools for publishers, press kits, and tracking | Leans toward a product ecosystem; still partial if the user wants a local agent skill | Good evidence that teams want operationalized launch prep |
| CapsuleGrader | Point solution for capsule art review | Fast, specific feedback on Steam capsule assets | Only covers art packaging, not page copy / demo / launch sequence | Useful sub-step, not the whole workflow |
| More Wishlists | Services + tooling | Steam-page analysis and professional services | More service-oriented than agent-reusable | Another signal that page optimization is its own niche |

## Categories
- **Official platform guidance**: Steamworks docs
- **Operator playbooks / educational guides**: How To Market A Game, presskit.gg, Indie Game Joe
- **Checklist + workflow SaaS**: IMPRESS
- **Point optimization tools**: CapsuleGrader, capsule/tag analyzers, wishlist calculators
- **Service-layer offerings**: More Wishlists and consulting-style audits

## What People Actually Use
In practice, teams stitch together docs, blog playbooks, and community advice. The recurring behavior is not "buy one tool and you're done." It is a manual stack:
1. read Steamworks docs for platform constraints,
2. study indie-marketing heuristics for wishlist and festival timing,
3. manually audit store assets/copy/tags,
4. prep a demo and outreach list,
5. ask peers/community for page feedback,
6. track wishlists and launch/festival milestones in a spreadsheet, checklist, or lightweight tool.

## Frequency Ranking
1. Steamworks docs / official platform surfaces
2. How To Market A Game / Chris Zukowski playbooks
3. Next Fest and store-page guides from presskit.gg / IMPRESS / similar specialist marketing sites
4. Community postmortems and Reddit discussions
5. Point tools for capsule art, tags, analytics, or wishlist estimation

## Key Gaps
- There is no local reusable skill in this repo that turns Steam-specific launch prep into a deterministic workflow.
- Existing solution space is fragmented across docs, blogs, SaaS checklists, and narrow point tools.
- Current catalog coverage jumps from broad game-dev planning (`bmad-gds`) to engine debugging (`game-build-log-triage`) without covering the Steam page / festival / launch-ops middle.
- The practical need is an audit-and-brief workflow, not a giant game-marketing encyclopedia.

## Contradictions
- Marketed claim: store-page optimization is a simple polish task.
  - User reality: it is entangled with tags, capsule hierarchy, trailers, demos, outreach timing, and launch sequencing.
- Marketed claim: festivals are just exposure events.
  - User reality: a weak or premature demo can waste the event and hurt perception.
- Marketed claim: marketing guidance is enough.
  - User reality: teams still need a reusable checklist that translates advice into concrete next actions.

## Key Insight
The best catalog move is not another broad game-marketing bundle. It is a focused Steam launch-ops skill that audits page readiness, wishlist funnel inputs, demo/festival timing, and outreach prerequisites in one repeatable brief. That fills a real gap between existing project-planning skills and current Steam-marketing point tools.
