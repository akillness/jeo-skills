# Solution Landscape: sprint-retrospective modernization

## Solution List
| Name | Approach | Strengths | Weaknesses | Notes |
|------|----------|-----------|------------|-------|
| Parabol | dedicated retro workflow | recurring cadence, action items, voting, async support | still needs external tracker ownership | strong dedicated reference point |
| TeamRetro | dedicated retro workspace | templates, voting, action tracking, team-health angle | can still become template-heavy | common remote-team tool |
| Retrium | facilitation-oriented retro product | coaching/facilitation depth | heavier than lightweight teams need | useful for facilitator patterns |
| EasyRetro / FunRetro / MetroRetro / Neatro | lightweight retro boards | easy sticky-note flow, anonymity, remote-friendly | follow-through often leaves the tool | good for board-mode reference |
| Miro / MURAL / FigJam | whiteboard templates | flexible hybrid collaboration | weak execution continuity by default | common “good enough” workaround |
| Atlassian Team Playbook / Jira / Confluence | guide + docs + tracker stack | fits existing Atlassian workflows | ceremony can become doc-only | strong default for software teams |
| Retromat / Scrum.org / Mountain Goat guides | facilitation kit / guidance | fresh prompts, format variety, facilitator education | not an execution system | useful reference layer |

## Categories
- Dedicated retrospective tools: Parabol, TeamRetro, Retrium, EasyRetro, FunRetro, MetroRetro, Neatro
- Whiteboard/template tools: Miro, MURAL, FigJam
- Tracker/doc stack substitutes: Jira, Confluence, Notion, Google Docs, Linear
- Facilitation pattern libraries: Retromat, Scrum.org, Mountain Goat Software, Atlassian Team Playbook

## What People Actually Use
In practice, teams mix a board or doc with a call, then push actions into a tracker. The real workflow is usually:
1. review previous actions
2. gather observations asynchronously or silently
3. cluster themes
4. vote or rank
5. create 1–3 owned actions with due dates
6. store summary in docs and execution in a tracker

## Frequency Ranking
1. Whiteboard/template tools (Miro, MURAL, FigJam)
2. Dedicated retro tools (Parabol, TeamRetro, Retrium, EasyRetro, FunRetro)
3. Atlassian-native workflows
4. Facilitation kits / prompt generators

## Key Gaps
- Action-item follow-through is usually weaker than note capture.
- Async and hybrid support still collapses back into manual facilitator work.
- Many tools optimize one session, not repeated-theme tracking across sprints.
- Milestone/postmortem variants for product and game teams are underrepresented in generic sprint-retro guidance.

## Contradictions
- Vendor pages imply the tool can own the workflow end to end, but teams still copy decisions into Jira, Linear, Notion, or spreadsheets.
- Template variety is marketed as the answer, but the deeper need is action review, pattern continuity, and facilitation boundaries.

## Key Insight
The highest-leverage modernization is not adding more retrospective templates. It is turning `sprint-retrospective` into a mode-selecting reflection skill that starts from prior commitments, supports live/async/hybrid facilitation, keeps action counts small and owned, and routes execution back into the team’s existing tracker.
