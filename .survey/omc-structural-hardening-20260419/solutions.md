## Frequency Ranking

Ranked by visible ecosystem prominence and relevance in current primary-source material, **not** install telemetry:

1. **OMC / oh-my-claudecode** — the canonical upstream wrapper and the most relevant evidence base for improving the `omc` skill itself ([repo](https://github.com/Yeachan-Heo/oh-my-claudecode), [README](https://github.com/Yeachan-Heo/oh-my-claudecode/blob/main/README.md), [releases](https://github.com/Yeachan-Heo/oh-my-claudecode/releases)).
2. **Claude Code native docs** — the baseline platform layer every wrapper ultimately depends on ([subagents](https://docs.anthropic.com/en/docs/claude-code/sub-agents), [hooks](https://docs.anthropic.com/en/docs/claude-code/hooks), [settings](https://docs.anthropic.com/en/docs/claude-code/settings)).
3. **wshobson/agents** — strong evidence for granular marketplace packaging instead of one giant orchestration surface ([repo](https://github.com/wshobson/agents)).
4. **oh-my-codex / OMX** — closest sister design; useful for portability and repeated failure-mode patterns across harnesses ([repo](https://github.com/Yeachan-Heo/oh-my-codex)).
5. **SuperClaude Framework** — popular adjacent framework, but more command/config heavy than OMC ([repo](https://github.com/SuperClaude-Org/SuperClaude_Framework)).
6. **Everything Claude Code** — important as a broad idea source, but too expansive to be the model for a bounded `omc` skill ([repo](https://github.com/affaan-m/everything-claude-code)).

## Categories

- **Canonical upstream wrapper**: OMC itself. Packaging pattern = marketplace plugin + npm/runtime + slash skills + CLI. Common pain = setup drift, duplicate surfaces, context bloat, state collisions.
- **Sibling cross-harness wrapper**: OMX. Packaging pattern = npm runtime + harness-native prompts/skills + tmux runtime + `.omx/` state. Common pain = the same operational hardening problems reappear on another host CLI.
- **Granular marketplace bundles**: `wshobson/agents`. Packaging pattern = many small plugins, each with narrow agents/skills. Common pain = discovery/install complexity and cache/plugin-name confusion.
- **Command/meta-frameworks**: SuperClaude. Packaging pattern = installer + many slash commands + optional MCP. Common pain = docs/version drift, indexing rituals, and operator complexity.
- **Full harness systems**: Everything Claude Code. Packaging pattern = repo + guides + multiple packages/apps/hooks. Common pain = surface sprawl and onboarding weight.
- **First-party platform primitives**: Claude Code native subagents/hooks/settings. Packaging pattern = docs and config, not a wrapper. Common gap = users still need routing conventions layered above the primitives.
