# Triage
- Problem: `code-review` is a high-utility legacy skill with broad generic wording, no support files, and weak boundaries to neighboring skills such as `git-workflow`, `debugging`, `testing-strategies`, and repo-management workflows.

- Why now: Recent graph and wiki runs show the code-quality cluster separating into sharper roles; `code-review` is still an unmodernized anchor and is likely creating activation noise and shallow reuse compared with recently upgraded neighboring skills.
