# Triage
- Problem: `task-estimation` is still a legacy generic agile skill with weak trigger boundaries, no support files, and shallow guidance compared with the modernized PM cluster.
- Audience: AI coding agents and operators doing backlog sizing, sprint planning prep, capacity/risk conversations, and delivery forecasting across software, product, and game work.
- Why now: `task-planning`, `standup-meeting`, and `sprint-retrospective` now explicitly route sizing work into `task-estimation`, so the estimation lane has become the weakest link in the project-management cluster and is a likely source of noisy activation or low-quality handoff behavior.
