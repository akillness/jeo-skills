# Triage
- Problem: `pattern-detection` is still a broad legacy skill centered on grep/regex snippets, so it blurs code scanning, security scanning, log/telemetry anomaly detection, and dataset outlier work without clear mode selection or route-outs.
- Audience: Agents and developers working across CLI/dev workflow, web/fullstack, PM/ops/marketing analytics, and game telemetry who need repeatable pattern/rule/anomaly hunting guidance.
- Why now: `data-analysis`, `log-analysis`, and `codebase-search` were recently modernized, which makes `pattern-detection` the highest-value remaining weak link in the search-analysis lane and the clearest opportunity to reduce overlap while adding references/evals.
