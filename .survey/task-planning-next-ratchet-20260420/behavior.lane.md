# Lane C — Actual Behavior

## What People Actually Use
- In practice, teams still pair one narrative planning surface with one tracking surface.
- Boards store tasks; docs/spreadsheets/meeting notes preserve readiness, blockers, and sequencing nuance.
- Game planning still needs discipline-aware milestone structure rather than pure scrum phrasing.

## Common Workarounds
- Plan in docs, then copy slices into Jira/GitHub/HacknPlan/other boards.
- Keep a ready/not-ready list outside the board.
- Use roadmap tools for direction and meetings/docs for last-mile execution translation.
- Explain dependencies verbally because the board is not enough.

## Pain Points With Current Solutions
- Boards become the default answer even when the missing thing is decomposition clarity.
- Discovery gets buried inside build tickets.
- Cross-functional work becomes an overgrown list.
- Planning artifacts swell until they duplicate estimation, board control, and review work.
