# Lane A — Context

This lane found that the best bounded next move is keeping `clawteam` tightly focused on the ClawTeam-specific operator surface: team/task/inbox/worktree state, board visibility, template launch, and profile/preset runtime setup. The real workflow context is operator-heavy, and adjacent repo skills already cover generic orchestration, planning, and board governance.

Key evidence:
- ClawTeam README quick-start and use-case flows center `team spawn-team`, `spawn`, `task`, `inbox`, `board`, `launch`, and `profile` / `preset` commands.
- Upstream issues show users still externalize due dates and face runtime truthfulness problems around worker persistence and config application.
- Repo-local boundaries already route generic orchestration outward.
