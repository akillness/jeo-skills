# Solution Landscape: bmad-idea modernization

## Solution List
| Name | Approach | Strengths | Weaknesses | Notes |
|------|----------|-----------|------------|-------|
| Miro / Mural / FigJam | Whiteboard-first workshop surface | Great for divergence, clustering, visual collaboration | Weak as long-term system of record; outputs still need manual synthesis | Most common workshop surface |
| Notion / Confluence / Coda / Airtable | Doc / database / brief templates | Async-friendly, searchable, easy to hand off | Can become generic note piles without method guidance | Common canonical artifact layer |
| Design Sprint / Double Diamond / IDEO Design Kit | Structured discovery process | Clear explore → converge flow | Facilitation-heavy; can feel workshop-theatrical | Useful method libraries, not one-stop routers |
| JTBD / Opportunity Solution Tree / Story Mapping | Opportunity and customer framing | Good for problem selection and experiment framing | Narrower than broader creative intake | Best for product/discovery lane |
| Strategyzer / Blue Ocean / positioning frameworks | Differentiation and market framing | Useful for offer/value/differentiation work | Often detached from delivery handoff if used alone | Best for GTM / strategy lane |
| StoryBrand / pitch/story frameworks | Narrative packaging | Good for buy-in, launch narrative, and pitch clarity | Risk of polished story without evidence | Best as a downstream packaging mode |
| TRIZ / Gamestorming / Liberating Structures | Creative problem-solving and facilitation methods | Strong divergence toolkits | Require method selection and active facilitation | Best as optional technique banks |

## Categories
- **Workshop platforms**: Miro, FigJam, Mural
- **Async structuring tools**: Notion, Confluence, Coda, Airtable
- **Discovery / design-thinking frameworks**: Design Sprint, Double Diamond, IDEO Design Kit
- **Customer / opportunity framing**: JTBD, Opportunity Solution Tree, Story Mapping
- **Strategy / differentiation**: Strategyzer, Blue Ocean, positioning work
- **Narrative / storytelling**: StoryBrand, creative brief, pitch/story frameworks
- **Creative problem-solving**: TRIZ, Gamestorming, Liberating Structures

## What People Actually Use
People rarely use one dedicated ideation system from start to finish. They usually combine boards, docs, chat, meetings, and decks. Whiteboards are transient; docs and briefs become canonical. In practice, the gap is not idea generation itself — it is choosing the right framing mode and producing one artifact that downstream planning or execution can actually use.

## Frequency Ranking
1. Whiteboard-first workshop tools (Miro / FigJam / Mural)
2. Docs / async brief systems (Notion / Confluence / Coda)
3. Design-thinking and discovery playbooks
4. JTBD / opportunity-framing methods
5. Strategyzer / positioning / differentiation frameworks
6. Narrative and pitch frameworks
7. Problem-solving facilitation libraries such as TRIZ / Gamestorming / Liberating Structures

## Key Gaps
- Very few solutions act as a **cross-domain creative router** that chooses between product framing, GTM framing, game concept framing, problem diagnosis, and narrative packaging.
- Many solutions optimize for workshop ideation but not for **one-artifact handoff** into planning, marketing execution, or game-production coordination.
- Users still need guidance on **which framing method to use next**, not just another canvas library.

## Contradictions
- Marketed promise: workshop tools make ideation easy.
- User reality: the real work still happens when someone converts scattered notes into a brief, a concept packet, or a planning handoff.

## Key Insight
The strongest bounded improvement is not to add another creativity wrapper. It is to modernize `bmad-idea` into the repository's **pre-planning idea router**: normalize the input, choose one operating mode, produce one concept artifact, and route explicitly into `bmad`, `task-planning`, `marketing-automation`, or `bmad-gds`.
